# ЗаписьИИ — ИИ-администратор для записи клиентов

Демо-продукт и кейс для портфолио, оформленный как **полноценная SaaS-панель**
(сайдбар-навигация, чат-центр, календарь, мини-CRM, метрики). ИИ-ассистент
общается с клиентом, подбирает услугу и время, **сам бронирует слот** в календаре
и шлёт напоминания. Универсальное демо: переключается между нишами (салон,
барбершоп, автосервис, стоматология) — услуги, цены и тон диалога меняются на лету.

## Зачем это бизнесу
- **67%** клиентов предпочитают писать, а не звонить (GfK, 2025).
- Главная боль — **пропущенные звонки** (ночью / в нерабочее время) и ошибки
  ручного администратора (двойные записи, забытые брони).
- Окупаемость в среднем **3–4 месяца**.

## Что внутри демо
- Живой чат со скриптовым «ИИ» (правила + сопоставление по ключевым словам) —
  работает **офлайн, без API-ключа**, ничего не сломается на показе клиенту.
- Понимает свободный текст: «постричься», «завтра», «в субботу», «16:00».
- FAQ-интенты в любой точке диалога: цены, адрес, часы работы, отмена/перенос —
  ИИ отвечает и возвращает клиента к текущему шагу записи.
- **Симуляция пропущенного звонка** (кнопка 📵 в шапке чата) — главный продающий
  сценарий: ночной недозвон → ИИ перехватывает и дозаписывает клиента.
- Кнопки быстрых ответов — диалог не заходит в тупик.
- **Месячный календарь-сетка**: занятые и прошедшие (на сегодня) слоты
  блокируются, у свободных дней — счётчик окошек; бронь занимает слот.
- Имитация СМС-напоминания (всплывает после записи).

### Разделы панели (сайдбар)
- **ИИ-ассистент** — чат с клиентом (главный экран).
- **Календарь** — месячная сетка + свободные слоты.
- **Записи** — мини-CRM: журнал броней глазами владельца, со счётчиком.
- **Клиенты** — база формируется автоматически из записей (имя, телефон, визиты).
- **Услуги** — прайс и длительность под текущую нишу.
- **Сценарии звонков** — запуск симуляции перехвата ночного недозвона.
- **Аналитика** — KPI (брони, сумма) + ROI-прикидка упущенной выручки.
- **Настройки** — переключение ниши, тумблеры уведомлений, точка обслуживания.

## Безопасность и надёжность
- Ввод пользователя рендерится только через `textContent` — XSS исключён
  (проверено инъекцией `<img onerror>`).
- Даты считаются в локальной таймзоне (не UTC) — нет сдвига дня по утрам.
- Валидация имени (только буквы) и телефона (10–12 цифр), ограничение длины ввода.

## Технологии
Чистый HTML / CSS / JS, без сборки и зависимостей. Один статичный сайт —
встраивается ссылкой в портфолио и открывается с телефона у клиента.

```
index.html   — каркас панели (сайдбар, чат, виды) + питч-секции
styles.css   — оформление дашборда (акцент меняется под нишу)
app.js        — движок диалога, календарь, мини-CRM, конфиг ниш (NICHES)
```

## Добавить новую нишу
В `app.js` дополнить объект `NICHES` блоком: `label`, `accent`, `name`, `role`,
`greeting`, `avgCheck`, `lostPerWeek` и список `services`
(`name`, `dur`, `price`, `kw` — ключевые слова для распознавания).

## Путь к продакшену (для реального клиента)
Скриптовый ИИ → заменить на **Claude API** (живой диалог) → подключить
**Telegram/WhatsApp** через бота → интеграция с реальным календарём
(YCLIENTS API / Google Calendar) → реальные напоминания (SMS/мессенджер).
